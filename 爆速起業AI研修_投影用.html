1-1 ／ 本講座のゴール

本講座でお伝えすること

1日1個ペースで新規事業を検証する「AI軍隊社員」の作り方

① 必ず知っておきたい、Claude Codeの基礎

非エンジニアでも迷わず使えるように、ゼロから丁寧に解説します。

② ゼロイチの作業を任せられる「AI社員」の作り方

ここが本丸です。具体的には以下の作業を全部AIに任せます。

  • 事業アイデアを生み出す市場リサーチ
  • 事業アイデアを事業計画書に起こして徹底シミュレーション
  • 事業の戦略整理・ブラッシュアップ
  • N=1分析からの、売れるコンセプト設計
  • 販売検証のためのLP作成から公開まで
  • MVPの作成と実装
  • 事業のセンターピン（KPI）の設計
  • 事業管理ダッシュボードの作成
あなたのやるべきことは一つだけ。 「目の前のお客さんに向き合うこと」
それ以外は、全部AIがやってくれます。

Claude Codeを使って「打席数」を爆上げしていきましょう💪
1-2 ／ Claude Code とは何か？

チャットツールとエージェント

チャットツールエージェント（Claude Code）
動き方一問一答計画 → 実行 → 振り返り
手足なしファイル・ブラウザ・コードを操作
自律性人間が都度指示指示なしで自律的に動く

エージェントの3つのポイント

1
計画する — タスクを分解し、順序を組み立てる
2
手足を持つ — ファイル・ブラウザ・コードを実際に操作する
3
振り返る — 結果を評価し、自律的に次の行動を決める

デモ構成

デモ① チャットとエージェントの違い（Claude in Chromeの実演）
デモ② ブラウザ版 Claude
デモ③ Claude Code で agents/cfo/AGENT.md を読ませた状態で同じ質問

2-3 ／ 事業アイデアをレビューする

事業アイデアをレビューする

作ったAI社員を使って、事業アイデアのブラッシュアップを始めます。

ドメインマスターAIでポジショニング分析

プロンプト例｜ドメインマスターAIを呼び出す agents/domain/AGENT.md を読んでください。そのロールとして振る舞い、以下の質問に答えてください。

「私の事業が参入しようとしている市場で、今の競合が埋めていない空白ポジションはどこですか？shared/knowledge/競合分析.md を参照して答えてください。」

戦略家AIで勝ち筋を言語化する

プロンプト例｜戦略家AI agents/strategist/AGENT.md を読んでください。戦略家として以下を分析してください。
shared/knowledge/競合分析.md と shared/assets/自分のアセット.md を参照して：

1. 私の事業が「なぜ勝てるか」を3類型（チャネル保有・未踏ポジション・市場規模型）で評価
　→ 該当する類型とその根拠を具体的に
　→ 該当しない類型は「なぜ当てはまらないか」も
2. 今の競合で誰も埋めていないポジションは何か
　→ 1年後に競合が追いかけてきたとき守れるか
3. ピボット案3パターン
　→ 今のアイデアが刺さらなかった場合の代替案を類型別に
　→ 各案にかかるピボットコスト（時間・お金）の概算も

参考：勝ち筋の3類型

類型一言定義なぜ勝てるか確認すること
チャネル保有型 すでに見込み顧客と接点がある
SNSフォロワー・メーリングリスト・既存顧客		 競合が同じサービスを作っても顧客に届かない。チャネルが参入障壁 チャネルの規模と質を数字で示す。フォロワー数より転換率が重要
未踏ポジション型 競合が埋めていない空白がある
ターゲットの細分化・価格帯の空白		 先行者優位が取れる。ただし「空白=需要なし」の可能性を常に疑う 空白の理由を答えられるか。意図的な空白か見落としかを区別する
市場規模型 市場が大きく量で勝てる
ニッチだが需要が安定している		 薄利多売でもスケールできる。AIで自動化すれば1人でも回せる 単価が低くてもLTVと継続率で補える設計になっているか

3-3 ／ TAM/SAMの算出

TAM/SAMの算出

「この事業アイデア、どれくらい伸びる可能性があるのか？」—— TAM/SAMは定義が重要。丸投げではなく、定義→シミュレーションのサポーターとして扱います。

STEP 1｜TAM/SAMシミュレーションの問いを立てる

プロンプト例｜CFO AI に実行させる CLAUDE.md と agents/cfo/AGENT.md を読んでください。shared/assets/自分のアセット.md も参照してください。

まず以下の2つだけ答えてください。

【1】TAM・SAM・SOMの違いを一言ずつ（3行で）

【2】私の事業に対して、TAMを計算するための「問いの立て方」を3パターン提示してください。

出力形式：
パターンA：（問いの切り口）
　→ 計算式のイメージ：
　→ このパターンが向いている理由：

パターンB・C も同様に

3パターン出し終えたら「A・B・Cのどれで数字を出しますか？」と聞いてください。数字はまだ出さないでください。

STEP 2｜TAM/SAMシミュレーション

プロンプト例｜数字を出す 選んだパターンで実際に数字を出してください。

出力形式：

SAM：絞り込み条件 / 計算式 / 信頼度
SOM（初年度）：根拠 / 目標人数・売上

3-4 ／ n=1の課題深掘り

n=1の課題深掘り

事業の立ち上げは、目の前の顧客の課題をとにかく具体的に、狂いなく特定することから始まります。戦略家AIを使って n=1 の課題を具体化していきましょう。

プロンプト例｜戦略家AIで深掘りする 戦略家として、私の想定ターゲット顧客の「最も痛い課題」を深掘りしてください。

手順：
1. まず私が想定するターゲット像を1人の具体的な人物として描写する
2. その人が「今すぐお金を払って解決したいこと」は何か
3. 「無料で代替できないか」を確認する。できるなら課題として弱い
4. 「3ヶ月後に解決されていないとどうなるか」を描写する
5. この課題、私のアセット（shared/assets/参照）で解決できるか評価する

課題の強度が弱い / アセットで解決できる可能性が低いものは、代替案を提案してください。
3-5 ／ コスト試算と最初の10人の集め方

コスト試算と最初の10人の集め方

初期ローンチに向けて「最小コストで検証できるか」を設計します。

STEP 1｜CFO AIで検証コストを試算する

プロンプト例｜CFO AI agents/cfo/AGENT.md を読んでください。CFO AIとして、この事業の「最小コスト検証シナリオ」を設計してください。

前提：調達なし（手元資金のみ）／ 1人でできる範囲 ／ 最初の10人を獲得するまで

出力：
1. 必要なコスト一覧（時間・金銭・機会コストを分けて）
2. 損益分岐点：何人売れれば初期コストが回収できるか
3. 「このコストで検証できる」の判断基準
4. できない場合：何を削ればできるようになるか
5.（オプション）融資検討時：日本政策金融公庫の新創業融資の概要

STEP 2｜営業AIで最初の10人獲得フローを設計する

プロンプト例｜営業AI・4週間フロー設計 agents/sales/AGENT.md を読んでください。営業AIとして、最初の10人を獲得する4週間のフローを設計してください。

・既存チャネル（SNS・LINE・人脈）をどう使うか
・各週の具体的なアクション（誰に・何を・どう言うか まで）
・「今週できる最初の1アクション」を今日中に決める
・コスト0円縛りでまず設計する（広告費は4週目以降）
・Claude の能力をフル活用して行うこと

3-6 ／ 月100万円シミュレーションとPPT出力

月100万円シミュレーションとPPT出力

最初の10人が獲得できた後、1年間で月100万円を達成できるかシミュレーションします。

プロンプト例｜戦略家AIでグロースプランを出す 戦略家として、最初の10人獲得後、月100万円の売り上げに到達するためのグロースプランを3パターン出してください。

パターン1：今の形を磨いて深く（質を上げる）
パターン2：横に広げる（類似顧客を増やす・チャネルを増やす）
パターン3：仕組み化してスケールする（AIでの自動化・外注）

各パターンについて：
・月100万円に到達する時期のめやす、3ヶ月後の売上目標
